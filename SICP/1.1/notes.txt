
Linguagem de programacao além de ser uma forma de se comunicar com o computador, é tambem um estrutura pela qual organizamos ideias sobre processos computacionais.

- Toda linguagem de programacao atual possui: 
    - expressoes primitivas
    - meios de combinacao: Elementos compostos são constituidos de elementos mais simples.
    - meios de abstracao: Elementos compostos pode ser nomeados e manipulados.

A linguagem possui dois elementos principais: procedimentos e dados.
Dados é aquilo que vocë deseja manipular e procedimentos é a regra para manipulacoa.

Expressao primitiva pode ser um numero

(3 * 5) + (10 - 6);
isso e uma combinacao

REPL = Leitura-avaliacao-impressao

A linguagem de programacao fornece meios para isso, um dos exemplos do livro foi:

Como o interpretador avalia combinacoes de operadores.
O intepretador faz isso ele faz uma avaliacao da combinacao em cada operando da combinacao
(2 + 4 * 6) * (3 + 12);

aqui temos (3+12),(4*6+2) e depois multiplicar.
Nessa questao ele traz a idéia de recursividade. entao percebe que o que ele faz e analisar as expressoes primitivas dessa combinacao.

Declaracao de funcoes.
operação composta pode receber um nome e, em seguida, ser chamada de unidade.


Para o quadrado perfeito de algo, retorna ele vezes ele mesmo

function square(x){
    return x*x;
}
Esse é um exemplo  de funcao composta
Onome é um símbolo a ser associado à definição da função no ambiente

Tendo square declarado podemos usar ele em uma expressao de "aplicacao de funcao"
square(21);
Posso usar ela entao, em blocos de construcao. 
Entao resumindo, eu tenho a funcao como: 
expressão-função-expressão(expressões-argumento)
a expressao-funcao é a expressao que representa a funcao
entao:
square(4)
o square é a expressao.. 
A funcao de aplicacao é a funcao resultante.
Para avaliar uma aplicação de função, faça o seguinte:

    Avalie as subexpressões da aplicação, a saber, a expressão de função e as expressões de argumento.
    Aplique a função que é o valor da expressão de função aos valores das expressões de argumento.

Bom entao primeiro ele avalia a expressao da funcao e o argumento, o valor do argumento é passado para funcao
expressões de aplicação de função também podem servir como expressões de argumento.
square(square(3)) = square(3) = 9 e square(9) é igual a 81.

Alem dessas funcoes compostas existem funcoes primitivas do propri js.
Como o interpretador avalia
1 - Avalia a expressao-funcao
2 - Avalia a empressao-argumento
3 - Avalia a expressao que representa a funcao.
4 - Avalia a expressao que representa o argumento.
5 - Aplica a funcao aos argumentos
Bom isso segue o modelo de substituicao em ordem de aplicativa.

1.1.6 Expressões Condicionais e Predicados´
function abs(x) {
  return x >= 0 ? x : - x;
}
ou seja, se x>=0 entao return x, se nao return  -x;
predicado é uma expressao que posso jugar verdadeiro ou falso.

function abs(x) {
  return x > 0
         ? x
         : x === 0
         ? 0
         : - x;
}

se x>0 return x, caso contrario x = 0l

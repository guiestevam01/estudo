Em geral a primeira etapa para LLM seria um pré trenamento, voce faz url-filtering, extrai textos de url, filtra textos, dominios,idiomas que voce quer nese dataset, no final voce tera algo como terabytes de texto bruto daquele filtro buscado pela internet. 
A proxima etapa é treinar redes neurais nisso. Antes precisa decidir como vamos representar este texto e como estamos alimentando.
Eles esperam uma sequencia unidimensional de simbolos e eles querem um conjunto finito de simbolos que sao possivel, portanto devemos decidir quais sao os simbolos e  entao representar nossos dados como sequencia unidimensional daqueles simbolos.
Entao temos uma sequencia unidimensional de texto. No geral obviamente estamos tratando de processamento computacdional entao cada simbolo desse texto é representado internamento como sequencia de bits, cada simbolo pode representar um conjunto de bits
Considerando grupo de bits consecutivos como 01111111 e agrupalos em um unico byte dicando tipo 124 86 105 etc.. sao valores para aquele grupo de bits em bytes, portanto temos uma melhor compressão do dataset bruto em bits que vao de 0 a 255, pense em cada byte como um ID ou simbolos unicos, a ideia e reduzir mais e comprimir mais essa sequencia, para isso podemos usar "The vite pair encoding algorithm" a ideia e procurar bytes conseuctivos que sao muito comum pense por exemplo em um conjunto finito unidimensional de bytes em grupo eles podem ter sequencias tipo 115 34 que pode ter ocorrencias de 94 vezes, entao vamos agrupo este par em um novo simbolo tipo, um id de 256 e vamos rescrever para cada  par 116 32 com este novo simbolo e entao iterar este algoritmo tenho algo como 100.000 simbolos possiveis gpt 4 usa 100.277 simbolos. Entao no geral ele faz tokens, imagina o conjunto [104, 101, 108, 108, 111, 32, 119, 111, 114, 108, 100] significa hello world utf8 o bpe olha para pares consecutivos como(104,101) → "he"
(101,108) → "el"
(108,108) → "ll", ele encontra os mais frequentes tipo "he" aparece MUITO em textos "ing" aparece MUITO (depois de algumas iterações) Ele cria um NOVO símbolo (token)

HTTP é um protocolo que permite obtencao de recursos web como doc hmtl. Ela utiliza a arquitetura cliente-servidor entao tudo parte da requisicao do cliente que geralmente seria um navegador web. Eles enviam request e recebe response do servidor. As request podem ser feitas por uma entidade, agente-usuario ou proxies em nome dele ai esse objeto de requisicao é solicitada pelo servidor que trata isso e envia uma response, no meio de todo esse processo existem entidades como proxies,gateways,caches...
O fluxo: 
1- o cliete abre uma conexao tcp, essa conexao sera usada para enviar uma ou varias request.
2 - o cliente envia a mensagem http como  GET / HTTP/1.1
Host: developer.mozilla.org
Accept-Language: fr 
3 - o servidor responde: HTTP/1.1 200 OK
Date: Sat, 09 Oct 2010 14:28:02 GMT
Server: Apache
Last-Modified: Tue, 01 Dec 2009 20:18:22 GMT
ETag: "51142bc1-7449-479b075b2891b"
Accept-Ranges: bytes
Content-Length: 29769
Content-Type: text/html

<!DOCTYPE html... (here comes the 29769 bytes of the requested web page)
mensagens de requisicao tem o metodo da requisicao, o path, e a versao do protocolo depois headers e corpo da entidaded.

metodos:
get - o cliente quer pegar algum recurso, o servidor quer disponibilizar o recurso
post - o cliente pode querer enviar dados para o servidor, o servidor quer receber dados enviado pelo client, post tambem cria novos dados.
delet - deletar dados
put - Se o recurso tem {nome: "João", idade: 30}
PUT precisa enviar o objeto completo {nome: "João", idade: 31} 
patch - altera parte do objeto

Nas solicitacoes temos get que no caso nao teria o corpo da entidade, mas em post como estamos criando dados voce pode enviar um corpo de entidade via json

1xx informacional
2xx sucesso: 200  OK, 201 CREATED
3xx Redirecionamento
4xx erro do cliente
5xx Erro no servidor
